## 🖱️ Cursor 核心技巧

### 1. Tab 键的妙用

Cursor 的 Tab 不仅仅是补全，它是**预测式编辑**：

```
普通补全：只填充当前行
Cursor Tab：可以预测你下一个编辑位置
```

**使用方式：**
- `Tab` - 接受建议并跳转到下一个预测位置
- `Ctrl+→` - 逐字接受建议
- `Esc` - 拒绝建议

### 2. 明确表达需求

❌ **模糊需求**（效果差）：
```
"实现一个文件上传功能"
```

✅ **明确需求**（效果好）：
```
"实现一个本地文件操作功能，只包含读取和写入，不包含上传到服务器"
```

### 3. 要求添加注释

在需求中明确要求 AI 添加注释：

```
"请编写一个 Python 爬虫，并添加详细注释解释每个函数的作用"
```

## ⚡ YOLO 模式

### 什么是 YOLO 模式？

YOLO 模式允许 AI 工具**不经确认直接执行**，形成"生成-测试-修正"闭环。

### 开启方式

1. `Ctrl+Shift+P` 打开命令面板
2. 输入 "YOLO"
3. 选择启用

### 效果对比

| 模式 | 代码可用率 |
|------|------------|
| 普通模式 | 35% |
| YOLO 模式 | 82% |

## 🔧 自定义模型配置

### 支持的模型

- GPT-5
- Claude 4.6 Sonnet/Opus
- GLM-5
- DeepSeek V3
- 所有 OpenAI 兼容协议模型

### 配置步骤

1. 打开 `Settings` → `Models`
2. 添加自定义模型名称
3. 填写 API Key
4. 修改 `Override OpenAI Base URL` 为实际地址

### 三种方案对比

| 方案 | 适用场景 | 配置难度 | 模型覆盖 |
|------|----------|----------|----------|
| 内置模型 | 只用 GPT-5/Claude 4.6 | ⭐ | 有限 |
| 官方 API Key | 想用自己的额度 | ⭐⭐ | 单厂商 |
| 聚合 API | 想一个 Key 切换 50+ 模型 | ⭐⭐⭐ | 最广 |

## 🎨 Composer 高级用法

### 什么是 Composer？

Composer 是 Cursor 的多文件编辑功能，可以同时修改多个文件。

### 使用场景

1. **重构多个文件**：同时修改多个相关文件
2. **添加功能**：跨文件添加新功能
3. **代码审查**：检查整个项目的代码质量

### 使用技巧

```
技巧：在测试和改 bug 阶段，先保存更改再接受

原因：给自己留下更多处理问题的空间
```

### 最佳实践

1. 使用 Composer 处理需要改多个文件的复杂任务
2. 先小范围测试，再大范围应用
3. 每次修改后及时测试
